# Render Deployment Copy

This folder is an isolated Render-ready copy of the Medical Student Tracker project. The original project files outside `ToRender/` were not modified for deployment.

## What is included

- `render.yaml` blueprint for a Render web service, PostgreSQL database, and persistent disk.
- `build.sh` to install dependencies, collect static files, run migrations, and run Django checks.
- `start.sh` to start Gunicorn.
- Render-ready `requirements.txt` with `gunicorn`, `whitenoise`, and `dj-database-url`.
- A copied Django app configured for Render PostgreSQL, WhiteNoise static files, and persistent media/backup paths.

## Deploy Steps

1. Create a new GitHub repository from the contents of this `ToRender/` folder, or push this folder as its own repo.
2. In Render, create a new Blueprint from `render.yaml`, or create a web service manually using these commands:
   - Build command: `./build.sh`
   - Start command: `./start.sh`
   - Health check path: `/health/ready/`
3. Ensure these environment variables are configured:
   - `DJANGO_ENV=prod`
   - `DJANGO_DEBUG=false`
   - `DJANGO_USE_SQLITE=false`
   - `DJANGO_SECRET_KEY` generated by Render
   - `DATABASE_URL` from Render PostgreSQL
   - `DJANGO_ALLOWED_HOSTS` only if you add custom domains. The app automatically trusts Render's built-in `RENDER_EXTERNAL_HOSTNAME`.
4. Configure email settings before using password reset:
   - `DJANGO_DEFAULT_FROM_EMAIL`
   - `DJANGO_EMAIL_HOST`
   - `DJANGO_EMAIL_PORT`
   - `DJANGO_EMAIL_HOST_USER`
   - `DJANGO_EMAIL_HOST_PASSWORD`
   - `DJANGO_EMAIL_USE_TLS=true` if your mail provider requires TLS
5. Keep the persistent disk mounted at `/var/lib/med-student-tracker` if you want uploaded workbooks, exports, and backups to survive redeploys.

## Important Notes

- Do not deploy with SQLite in production. This copy expects Render PostgreSQL through `DATABASE_URL`.
- Render disks are tied to one service and region. For a more robust production setup, consider object storage for uploads and external backups.
- The app stores uploaded workbooks and backup files under `/var/lib/med-student-tracker` in this Render copy.
- Run `python manage.py createsuperuser` from the Render shell after first deploy if you need an initial admin account.
